About Me

사용자 흐름을 중심으로 설계하고, 감성과 구현 가능성을 함께 봅니다.

보기 좋은 화면보다 먼저, 사용자가 쉽게 이해하고 자연스럽게 이동하는 경험을 설계합니다.

감성적인 비주얼과 명확한 구조를 함께 만들고, Figma에서 시작한 디자인을 실제 웹 구현까지 연결할 수 있는 것이 저의 강점입니다.

Design Perspective 사용자 흐름을 중심으로 화면을 설계하며, 첫인상보다 사용 과정 전체가 자연스럽게 이어지는 구조를 중요하게 봅니다.
User Experience 사용자 경험은 단순한 동선이 아니라 이해 속도와 감정까지 포함된다고 생각하며, 직관성과 몰입감을 함께 고려합니다.
Design + Publishing Figma에서 설계한 화면을 HTML5, CSS3, JavaScript로 연결할 수 있어 디자인과 퍼블리싱 사이의 간극을 줄일 수 있습니다.
Collaboration 기획 의도를 시각적으로 정리하고, 구현 가능성을 함께 검토하며 협업하는 방식을 선호합니다. 그래서 실무에서 더 매끄럽게 연결되는 결과물을 만드는 데 강점이 있습니다.

Case Study

왜 이렇게 설계했는지와 어떤 판단으로 문제를 풀었는지를 빠르게 읽을 수 있도록 정리한 가상 프로젝트 사례입니다.

Case Study 01

뷰티 커머스 앱 온보딩 및 상품 탐색 경험 개선

신규 사용자가 앱의 강점을 빠르게 이해하지 못하고 첫 탐색 단계에서 이탈하는 문제를 가정했습니다. 온보딩과 카테고리 진입 구조를 재설계해, 서비스 이해도와 첫 탐색 전환율을 함께 높이는 방향으로 설계했습니다.

프로젝트 개요 모바일 뷰티 커머스 앱의 첫 진입 경험 전면 개선
문제 정의 서비스 강점이 분산되어 첫 화면에서 핵심 가치가 전달되지 않음
목표 온보딩 이해도 향상, 카테고리 진입률 증가, 첫 저장 행동 유도
타깃 사용자 성분과 후기 비교에 익숙하지만 탐색 피로도가 높은 20~30대 사용자

리서치 / 인사이트

사용자는 모든 데이터를 보기보다, 핵심 지표와 이상 신호를 빠르게 읽고 싶어 했습니다.

정보 구조

상단 KPI, 중단 추세 그래프, 하단 세부 테이블의 3단 구조로 우선순위를 재정렬했습니다.

와이어프레임 / UI

스캔 동선을 기준으로 카드와 필터를 정리하고, 시각적 대비로 인지 부하를 낮췄습니다.

프로토타입 / 산출물

KPI 카드, 필터 구조, 그래프 컴포넌트, 상태 배지를 포함한 대시보드 UI 가이드를 완성했습니다.

성과 / 회고 핵심 가치를 먼저 보여주는 구조가 첫 인상을 바꾼다

기능이 많은 서비스일수록 무엇을 먼저 보여줄지 결정하는 정보 설계가 전환율에 직접적인 영향을 준다고 판단했습니다.

설계 포인트 온보딩을 설명이 아닌 행동 유도 흐름으로 재정의

기업 관점에서는 단순 미적 개선이 아니라, 이탈 구간을 명확하게 줄이기 위한 구조적 설계라는 점이 핵심입니다.

기업 어필 포인트 문제 정의부터 UI까지 일관된 논리로 연결

리서치 인사이트를 정보 구조와 디자인 의사결정으로 이어서 설명할 수 있다는 점이 강점입니다.

Case Study 02

B2B SaaS 대시보드의 복잡한 데이터 가시성 개선

운영 지표가 많은 B2B 대시보드에서 사용자가 중요한 데이터를 빠르게 찾지 못하는 문제를 가정했습니다. 우선순위가 높은 지표를 먼저 이해하고 의사결정까지 이어질 수 있도록 레이아웃과 정보 구조를 재정비했습니다.

프로젝트 개요 운영 지표 중심 SaaS 대시보드의 정보 구조 및 UI 개선
문제 정의 지표는 많지만 우선순위가 불명확해 실무 의사결정에 시간이 오래 걸림
목표 핵심 KPI 가독성 향상, 필터 사용성 개선, 보고 시간 단축
타깃 사용자 운영팀 리드, 마케터, 데이터 기반으로 빠르게 판단해야 하는 실무 담당자

리서치 / 인사이트

사용자는 모든 데이터를 보기보다, 핵심 지표와 이상 신호를 빠르게 읽고 싶어 했습니다.

정보 구조

상단 KPI, 중단 추세 그래프, 하단 세부 테이블의 3단 구조로 우선순위를 재정렬했습니다.

와이어프레임 / UI

스캔 동선을 기준으로 카드와 필터를 정리하고, 시각적 대비로 인지 부하를 낮췄습니다.

프로토타입 / 산출물

KPI 카드, 필터 구조, 그래프 컴포넌트, 상태 배지를 포함한 대시보드 UI 가이드를 완성했습니다.

성과 / 회고 데이터 UI는 미려함보다 우선순위 설계가 핵심이다

정보를 많이 담는 것보다 무엇을 먼저 읽게 할지 정하는 구조적 판단이 대시보드 품질을 좌우한다고 보았습니다.

설계 포인트 지표의 양을 줄인 것이 아니라, 읽는 순서를 설계

기업 관점에서는 화면을 예쁘게 만든 것이 아니라 실무 판단 속도를 높이기 위한 UX 개선이라는 점을 분명히 보여줍니다.

기업 어필 포인트 복잡한 정보를 구조화해 실행 가능한 화면으로 정리

데이터가 많은 환경에서도 UX 논리와 UI 디테일을 함께 설계할 수 있다는 점이 경쟁력이 됩니다.
